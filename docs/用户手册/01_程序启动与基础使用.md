# 程序启动与基础使用

## 1. 程序放置位置

实验室电脑上的程序目录通常放在：

```bash
~/proj/Recordlab_host
```

日常使用请在这个目录下启动，不要从开发仓库路径启动。

## 2. 如何启动程序

在终端进入程序目录后执行：

```bash
cd ~/proj/Recordlab_host
./RecordLabHost.sh
```

首次启动或环境损坏时，程序可能会自动安装依赖、自动重新编译。这个过程可能持续几分钟，属于正常现象。

## 3. 自动更新说明

每次运行 `./RecordLabHost.sh` 时，程序会先尝试：

1. 免密连接服务器 `nreal@10.2.11.200`
2. 从 `/home/nreal/nviz_record_data/Recordlab_host` 同步最新程序
3. 检查本机依赖是否完整
4. 检查本机二进制是否需要重新构建
5. 打开主界面

如果自动更新失败，程序会弹出提示，并继续尝试启动本机现有版本。

## 4. 主界面基础概念

- `选择主 Agent`：选择当前要使用的设备节点。
- `脚本执行`：推荐用户优先使用这里的现成脚本。
- `数据 + 命令`：查看数据、执行基础命令。
- `虚拟节点`：仅供开发人员使用，普通用户不要点击。

## 5. 推荐使用流程

1. 启动程序。
2. 在首页选择本次实验要用的主 Agent。
3. 等待进入主页面。
4. 优先在 `脚本执行` 页运行默认脚本。
5. 如需手动控制设备，再到 `数据 + 命令` 页执行命令。
6. 实验结束后确认录制停止，再关闭程序。

## 6. 常用命令含义

在 `数据 + 命令` 页里，常见命令含义如下：

- `init_device`：初始化设备连接。
- `start_device`：开始设备数据流或工作流程。
- `stop_device`：停止设备数据流或工作流程。
- `start_record`：开始录制。
- `stop_record`：停止录制。
- `release_device`：释放设备连接。
- `reboot_device`：重启设备。

普通实验优先使用脚本，不建议手动随意组合命令。

## 7. 如何判断程序是否正常

以下现象通常表示程序运行正常：

- 状态栏显示了版本号。
- 已经选择到正确的主 Agent。
- `脚本执行` 页能看到脚本列表。
- `数据 + 命令` 页日志持续刷新。
- 传感器页面不再长时间停留在“等待数据...”。

## 8. 常见问题与处理

### 8.1 启动后提示无法免密连接服务器

含义：

- 自动更新没有成功。

处理：

1. 先点掉弹窗。
2. 程序通常仍会继续启动本机版本。
3. 如果多台机器都出现这个问题，联系开发人员检查服务器或免密配置。

### 8.2 启动很慢，停留在安装依赖或构建阶段

含义：

- 本机 Python 环境损坏，或二进制需要重新构建。

处理：

1. 先等待几分钟。
2. 如果长时间不结束，记录终端输出并联系开发人员。

### 8.3 界面打开了，但没有数据

处理：

1. 检查主 Agent 是否选对。
2. 检查设备是否正确连接。
3. 检查是否已经运行了对应脚本或执行了必要命令。
4. 如果是眼镜/手机设备，先断开再重新连接。

### 8.4 `init_device` 失败

处理：

1. 拔掉设备后重新插入。
2. 等待界面状态恢复，再重新尝试。
3. 如果是手机类设备，检查手机与电脑网络连接是否正常。

### 8.5 录制停止后文件没看到

处理：

1. 先确认脚本流程已经走完。
2. 再检查日志里是否有 `stop_record` 成功提示。
3. Nebula、批处理类脚本有时收尾较久，请多等待一会儿。

## 9. 用户注意事项

- 不要点击 `虚拟节点` 页。
- 不要运行名字明显带 `test` 的测试脚本，除非开发人员明确要求。
- 不要手动删除程序目录下的 `third_party/`、`build/`、`.venv-py310/`。
- 出现问题时，优先保留终端输出和截图，不要反复强制重启。
