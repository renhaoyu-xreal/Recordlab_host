# NViz 使用手册

## 1. 适用场景

本手册对应主 Agent：

- `glasses_nviz_node`

适用于 NViz 相关采集与录制。

## 2. 进入方式

1. 启动 `./RecordLabHost.sh`
2. 在首页选择 `glasses_nviz_node`
3. 进入主页面后，优先使用 `脚本执行` 页

不要点击 `虚拟节点` 页。

## 3. 推荐脚本

常用默认脚本：

- `nviz/record_nviz_simple.py`：基础 NViz 录制
- `nviz/record_ur_gt_3dof_batch.py`：NViz 与 UR 联动批量录制

名字带 `test` 的脚本不是给普通用户日常实验使用的。

## 4. 推荐操作流程

1. 选择 `glasses_nviz_node`
2. 在 `脚本执行` 页选择合适脚本
3. 启动脚本
4. 观察脚本流程和日志
5. 等待录制结束
6. 确认脚本停止后再关闭程序

## 5. 手动命令流程

如果开发人员要求手动执行命令，推荐顺序：

1. `init_device`
2. `start_device`
3. `start_record`
4. `stop_record`
5. `stop_device`
6. `release_device`

不要跳过 `stop_record` 直接断开程序。

## 6. 正常现象

- 启动后能看到 IMU 数据刷新。
- 录制开始附近可能出现一次短暂数据切换。
- 录制停止后需要等待一会儿才能完全收尾。

## 7. 常见问题

### 7.1 `init_device` 失败

处理：

1. 断开眼镜。
2. 等待界面恢复。
3. 重新插入眼镜。
4. 再次尝试。

### 7.2 脚本启动了，但看不到数据

处理：

1. 确认主 Agent 是 `glasses_nviz_node`
2. 确认设备连接稳定
3. 确认脚本不是测试脚本

### 7.3 录制开始时界面有短暂波动

这通常是正常现象。当前 NViz 录制开始时会主动对齐录制起点，开始录制附近可能有一次短暂切换。

## 8. 用户注意事项

- 优先用默认脚本，不要自己拼命令。
- 不要点击 `虚拟节点`。
- 如果是 UR 联动脚本，务必确认机械臂环境和安全区已准备好。
